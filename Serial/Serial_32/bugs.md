# 问题记录

- 问题：`Serial.c` 第 1 行存在裸文本 `serial.c`。
  - 根因：文件名文本被写入源码。
  - 影响：C 编译会直接失败。
  - 状态：已修复。

- 问题：`Serial.c` 引用 `serial.h`，但目录内缺少该头文件。
  - 根因：接口头文件未提交或未生成。
  - 影响：编译时无法解析宏、全局变量和函数声明。
  - 状态：已修复，已新增 `include/serial.h`。

- 问题：串口 1 接收长度没有按实际缓冲区上限检查。
  - 根因：`RxData1 <= 255` 对 `uint8_t` 恒成立。
  - 影响：异常长度帧可能导致接收缓冲越界。
  - 状态：已修复，长度检查改为基于 `MAX_FRAME_LEN` 的条件分支。

- 问题：串口 1 解析 16 位数据时未做有符号扩展。
  - 根因：高低字节拼接后直接赋给 `int32_t`。
  - 影响：负数数据会被解析为正数。
  - 状态：已修复，先拼接为 `uint16_t` 后转换为 `int16_t`。

- 问题：串口 1 接收索引用 `uint8_t` 表示。
  - 根因：原实现未定义 `MAX_FRAME_LEN`，索引类型没有覆盖 256 字节帧。
  - 影响：当长度为 255 时，长度字节加数据区共 256 字节，索引可能溢出。
  - 状态：已修复，接收索引改为 `uint16_t`。

- 问题：真实 STM32 HAL 工程构建未验证。
  - 根因：当前 `Serial_32` 目录没有完整 STM32CubeMX/STM32F1 HAL 工程、芯片型号头文件和构建系统。
  - 影响：只能验证当前 C 文件在 HAL 桩下的语法正确性，不能确认目标板完整固件构建结果。
  - 状态：未验证。

- 问题：README 缺少 `Serial_32` 说明。
  - 根因：`package/communication/README.md` 原本为空文件。
  - 影响：目标芯片、帧格式、验证范围和集成前置条件不清晰。
  - 状态：已修复，已补充 STM32F103C8T6 相关说明。
